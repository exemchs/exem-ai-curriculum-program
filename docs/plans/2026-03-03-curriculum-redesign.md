# EXEM AI Curriculum Redesign

설계: 조현서, CX그룹장 @ EXEM
기준일: 2026.03.03

## 배경

기존 커리큘럼의 문제:
- 설치 안내가 랜딩 페이지와 중복
- 텍스트 나열만으로 시각적 재미 없음
- Cursor/터미널 기초 설명 부족
- Day 2~5 콘텐츠가 EXEM 실무와 연결 약함
- 전체적으로 얕고 "읽는 교재" 느낌

## 대상

EXEM CX그룹 비전공자 위주. ChatGPT/Claude 웹은 써봤지만 Claude Code는 처음.

## 목표

5일 후 참가자가:
- Claude Code를 자유롭게 쓸 수 있고
- 본인 업무에 실제 적용할 Skill/워크플로우를 1개 이상 만든다

## 핵심 내러티브

"프롬프트를 잘 쓰는 사람이 아니라, 컨텍스트를 잘 설계하는 사람이 되어라."

기존: "기능을 하나씩 배운다" (설치 → MCP → Skill)
변경: "레벨업하며 시스템을 쌓는다" (질문법 → 실무 → 도구 → 자동화 → 완성)

- Ethan Mollick (Wharton 교수) AI 위임 프레임워크

## 디자인 시스템

### 시각 규칙

- Day 시작: EXEM ASCII 아트 배너 (Day 0만) / `═══` 헤더 (Day 1~5)
- 섹션 헤더: `[N] 제목` + `═══` (굵은 선)
- 섹션 끝: `───` (가는 선)
- 닫힌 박스 금지: 한글 정렬 깨짐 방지
- 일러스트: 화면/아키텍처 등 "보여줘야 이해되는 것"에만 (영어 위주, 한글 설명은 바깥에)
- 아이콘: 비교, 흐름, 팁 등에 활용
- 프로그레스: 체크리스트 + `████░░░░` 바
- 인터랙션: 매 단계 AskUserQuestion
- 설치 안내: 프로그램에서 반복하지 않음. 랜딩 페이지로 리디렉션

### Day 1: AI에게 잘 시키는 법 (60분)

핵심 메시지: "프롬프트를 잘 쓰는 시대는 끝났다. 시스템을 설계하라."

| Block | 주제 | 시간 | 핵심 |
|-------|------|------|------|
| 1 | 왜 Claude Code인가 | 15분 | 웹 Chat vs Claude Code 직접 비교 체험. 같은 작업을 두 방식으로 해보고 차이 체감 |
| 2 | AI에게 잘 시키는 3가지 레벨 | 15분 | Lv.1 구조화된 요청(CO-STAR) → Lv.2 AI가 역질문(CRIT Interview) → Lv.3 시스템에 새기기(CLAUDE.md 예고) |
| 3 | CLAUDE.md — 내 비서 세팅 | 20분 | 이름, 역할, 선호 규칙 직접 작성. "한번 쓰면 영구 기억" 체험 |
| 4 | 내 업무 하나 해보기 | 10분 | 배운 것을 적용해서 실제 업무(보고서/정리/요약) 하나 처리 |

### Day 2: 내 업무를 AI로 해보기 (60분)

핵심 메시지: "AI는 도구가 아니라 역량(capacity)이다."

| Block | 주제 | 시간 | 핵심 |
|-------|------|------|------|
| 1 | 위임의 기술 | 15분 | Ethan Mollick의 3분류: Just Me / Delegate / Automate. 내 업무를 분류해보기 |
| 2 | Delegate 실습 | 25분 | "좋은 결과가 뭔지 정의하고, AI에게 맡기고, 검증한다" 실제 업무로 실습 |
| 3 | AI의 한계 알기 | 20분 | 환각(hallucination), 맥락 한계, 인터넷 접근 불가 등. 실패 체험 후 대응법 |

### Day 3: 도구 연결하기 — MCP (60분)

핵심 메시지: "AI의 능력이 부족한 게 아니라, 맥락이 부족한 것이다."

| Block | 주제 | 시간 | 핵심 |
|-------|------|------|------|
| 1 | MCP 개념 | 10분 | "AI에게 눈과 손을 달아주는 것." USB-C 비유. 아키텍처 일러스트 |
| 2 | 실제 연결 | 25분 | Notion 또는 Google Calendar MCP 연결 실습 |
| 3 | 연결된 도구로 업무하기 | 25분 | MCP로 가져온 데이터로 실제 업무 처리 체험 |

### Day 4: 자동화하기 — Skill + 검증 루프 (60분)

핵심 메시지: "한 번 잘 시키는 게 아니라, 반복 가능하게 만들어라."

| Block | 주제 | 시간 | 핵심 |
|-------|------|------|------|
| 1 | Skill이란 | 10분 | "반복 업무를 /명령어로 만드는 것." 예시 시연 |
| 2 | 내 Skill 만들기 | 30분 | 본인 업무 기반 Skill 직접 제작 (SKILL.md 작성) |
| 3 | 검증 루프 설계 | 20분 | "AI가 스스로 확인하게 하면 품질 2~3배." Skill에 검증 단계 추가 |

### Day 5: 졸업 (60분)

핵심 메시지: "'소프트웨어 엔지니어'가 아니라 '빌더'의 시대."

| Block | 주제 | 시간 | 핵심 |
|-------|------|------|------|
| 1 | 5일간 배운 것 정리 | 10분 | CLAUDE.md + MCP + Skill + 검증 루프 = 나만의 AI 시스템 |
| 2 | 졸업 프로젝트 완성 | 30분 | Day 4에서 만든 Skill을 고도화하거나 새로운 워크플로우 완성 |
| 3 | 팀 공유 + 마무리 | 20분 | 만든 것을 팀에 공유하는 방법 안내. 커리큘럼 마무리 |

## 진행 방식

### Phase A/B 프로토콜 (Day 1~5)

각 Block은 2턴에 걸쳐 진행:

Phase A (첫 번째 턴):
1. reference 파일의 EXPLAIN 섹션을 읽고 설명
2. 시각 요소(일러스트/아이콘/다이어그램) 포함
3. reference 파일의 EXECUTE 섹션을 읽고 실행 안내
4. AskUserQuestion으로 진행 선택지 제공
5. STOP

Phase B (두 번째 턴):
1. reference 파일의 QUIZ 섹션을 읽고 AskUserQuestion으로 퀴즈
2. 피드백 후 다음 Block으로

Day 0은 퀴즈 없이 단계별 AskUserQuestion으로 진행.
