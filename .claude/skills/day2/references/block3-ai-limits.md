## [3] AI를 잘 쓰려면 알아야 할 것
═══════════════════════════════

2026년의 AI는 놀라울 정도로 잘합니다.
웹 검색도 되고, 파일도 읽고, 긴 대화도 기억합니다.

그런데 여전히 실수합니다. 어떤 상황에서?

### AI가 잘하는 것 vs 확인이 필요한 것

핵심 원칙: **직접 확인할 수 있는 것은 정확, 기억에 의존하는 것은 불확실**

  ✅ AI가 잘하는 것 (높은 신뢰)
  ─────────────────────────────
  파일 읽고 요약하기       → 직접 읽으니까 정확
  코드 분석하기            → 직접 보니까 정확
  형식 변환하기            → 규칙 기반이라 정확
  웹 검색 후 정리          → 출처가 있으니까 확인 가능

  ⚠️ 확인이 필요한 것 (과신 주의)
  ─────────────────────────────
  특정 회사의 수치/통계     → 학습 데이터에 없으면 지어냄
  내부 정보/비공개 데이터   → 접근할 수 없는 건 추측함
  미묘한 뉘앙스/정치적 판단 → 사람의 판단이 필요
  "항상" "절대" 같은 단정   → 예외가 있을 수 있음


### 가장 위험한 순간: 과신 (Over-confidence)

AI의 진짜 위험은 "못하는 것"이 아닙니다.
**틀린 답을 자신있게 말하는 것**이 위험합니다.

  😰 위험한 상황
  ─────────────────────────────
  "EXEM의 2024년 매출은 612억원입니다."
  → 맞을 수도 있고, 지어낸 것일 수도 있음
  → 문제는 AI가 둘 다 같은 확신으로 말한다는 것

  👍 안전한 사용
  ─────────────────────────────
  "EXEM에 대해 알고 있는 것을 알려줘.
   확실하지 않은 정보는 '불확실'이라고 표시해."
  → AI에게 불확실성을 표시하도록 명시적으로 요청


### 내 도구/데이터에 접근하기

AI가 웹 검색은 할 수 있지만,
**당신의 Notion, Calendar, Slack 데이터**에는 접근할 수 없습니다.

이것이 Day 3에서 MCP를 배우는 이유입니다:
  MCP 없이 → "내 캘린더 일정 알려줘" → 불가능
  MCP 연결 → "내 캘린더 일정 알려줘" → 바로 실행

### AI의 과신 체험

직접 체험해봅시다.

**체험 1: 과신 탐지**
아래를 입력해보세요:
```
EXEM의 2025년 4분기 매출을 알려줘.
```
→ Claude의 답을 보세요. 자신있게 답하나요?
→ 그 수치가 정확한지 확인할 수 있나요?

**체험 2: 불확실성 표시 요청**
이번엔 이렇게 입력해보세요:
```
EXEM에 대해 알고 있는 것을 알려줘.
확실한 정보와 불확실한 정보를 구분해서 표시해줘.
```
→ 체험 1과 비교해보세요. 어떻게 다른가요?

**체험 3: 검증 가능한 요청**
이번엔 AI가 잘하는 종류의 요청:
```
이 프로젝트의 CLAUDE.md를 읽고 핵심 내용을 5줄로 요약해줘.
```
→ 직접 파일을 읽으므로 정확합니다. 차이를 느껴보세요.

💡 핵심 습관:
   "AI가 직접 확인할 수 있는 요청" vs "기억에 의존하는 요청"을 구분하세요.
   후자에는 항상 검증 단계를 추가하세요.
