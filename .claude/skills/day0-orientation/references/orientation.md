# Day 0 오리엔테이션


## [1] 이건 결국 대화입니다
═══════════════════════════════

ChatGPT나 Claude 웹에서 대화해본 적 있죠?
Claude Code도 그것과 동일합니다. 질문하면 답하고, 부탁하면 실행합니다.

솔직히 요즘 웹 AI도 파일 업로드, MCP 연결, 코드 실행까지 다 됩니다.
그럼 Claude Code를 왜 쓰냐?

  💡 핵심 차이: "내 맥락이 쌓이는 AI"

  CLAUDE.md    → 내 규칙과 선호를 저장하는 AI 업무 매뉴얼
  /명령어       → 반복 업무를 한마디로 자동화 (Skill)
  플러그인      → 다른 사람이 만든 워크플로우를 가져다 쓸 수 있음

  웹 AI는 매번 새로 시작하지만,
  Claude Code는 내 맥락이 쌓일수록 더 똑똑해집니다.

지금 이 대화 자체가 Claude Code입니다. 이미 시작한 겁니다.

## [2] Cursor 안에서 뭘 보고 있는 건지
═══════════════════════════════

지금 보고 있는 화면이 Cursor입니다.
3곳만 기억하세요:

```
+------------------+------------------------+
|                  |                        |
|   EXPLORER       |       EDITOR           |
|                  |                        |
|   📁 .claude/    |   (file content here)  |
|   📁 docs/       |                        |
|   📄 CLAUDE.md   |                        |
|   📄 README.md   |                        |
|                  |                        |
+------------------+------------------------+
|                                           |
|              TERMINAL                     |
|   > You are talking to Claude here        |
|                                           |
+-------------------------------------------+
```

왼쪽: Explorer (파일 탐색기) — 프로젝트 파일 목록
가운데: Editor (에디터) — 파일 내용 표시
아래: Terminal (터미널) — Claude Code가 동작하는 곳 ← 지금 여기

💡 파일 탐색기가 안 보여도 괜찮습니다!
   지금은 터미널 대화에 집중하세요.
   나중에 파일을 눈으로 보고 싶으면
   Cursor 메뉴 → File → Open Folder → exem-ai-curriculum-program 선택
   (이때 터미널이 초기화되므로 Claude 세션이 끝난 후에 하세요)

## [4] 멈추기 / 이어하기
═══════════════════════════════

💬 멈출 때:

  1. "여기까지 정리해줘" 입력
  2. Claude가 진행 상황 저장 💾
  3. 터미널을 닫아도 OK 👋

🔄 이어할 때:

  1. Cursor 열기 → File → Open Folder → exem-ai-curriculum-program 선택
  2. 터미널 열기 (macOS: Cmd+J / Windows: Ctrl+J)
  3. `claude` 입력
  4. "이어서 해줘" 입력

언제든 멈추고, 언제든 이어할 수 있습니다.
