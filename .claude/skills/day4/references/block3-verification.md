## [3] 검증 루프 설계
═══════════════════════════════

Boris Cherny(Claude Code 창시자)의 핵심 원칙:
"Claude에게 스스로 검증할 수단을 주면 품질이 2~3배 올라갑니다."

### 검증 루프란?

AI가 결과를 내놓은 후, 스스로 확인하는 구조를 Skill에 내장하는 것입니다.

비유: 요리사의 맛보기

  🍳 맛보기 없는 레시피            👨‍🍳 맛보기 있는 레시피
  ─────────────                  ─────────────
  재료 넣고 → 완성                 재료 넣고 → 맛보기
  결과가 어떨지 모름               → 간 조절 → 완성
  운에 맡김                       품질 보장

레시피에 "맛보고 간 조절" 단계를 넣는 것처럼,
Skill에 "결과를 확인하고 보완" 단계를 넣습니다.


### 3가지 검증 방법

  🔍 자가 검증
  ─────────────────────────────
  "결과를 다시 읽고 누락된 항목이 있는지 확인해"
  가장 기본적인 검증. 모든 Skill에 넣을 수 있음.

  ✅ 체크리스트 검증
  ─────────────────────────────
  "다음 기준을 모두 충족하는지 확인:
   - 항목이 5개 이상인가?
   - 출처가 포함되어 있는가?
   - 3문장 이내로 요약되어 있는가?"
  구체적인 기준이 있을 때 효과적.

  🔄 역검증
  ─────────────────────────────
  "이 보고서를 받는 팀장 입장에서
   빠진 정보가 없는지 점검해"
  관점을 바꿔서 검증. 가장 높은 수준.


### Skill에 검증 루프 추가하는 법

SKILL.md에 검증 단계를 추가합니다:

```
## 진행 순서
1. [업무 실행]
2. [결과 생성]
3. 결과를 다시 읽고 다음을 확인:
   - [확인 항목 1]
   - [확인 항목 2]
   - [확인 항목 3]
4. 문제가 있으면 수정 후 다시 확인
5. 최종 결과 출력
```

## EXECUTE

### Block 2에서 만든 Skill에 검증 추가

**1단계: 검증 기준 정하기**
내 Skill의 "좋은 결과" 기준을 3가지 정합니다:
- 형식 기준: "표 형식인가?", "3줄 이내인가?"
- 내용 기준: "핵심 항목이 빠지지 않았는가?"
- 품질 기준: "그대로 제출할 수 있는 수준인가?"

**2단계: Skill에 검증 단계 추가**
Claude에게 요청합니다:
```
아까 만든 /[내스킬이름] Skill의 SKILL.md를 수정해줘.
마지막 단계에 검증 루프를 추가해:

결과를 다시 읽고 다음을 확인:
- [기준 1]
- [기준 2]
- [기준 3]
문제가 있으면 수정하고 다시 확인해.
```

**3단계: 검증 전/후 비교**
같은 Skill을 두 번 실행합니다:
1. 검증 루프 없는 버전 (이전 결과를 기억해두세요)
2. 검증 루프 있는 버전 (방금 수정한 것)

💡 검증 루프가 추가되면:
   AI가 스스로 빠진 부분을 찾아서 보완합니다.
   사람이 일일이 확인하지 않아도 품질이 올라갑니다.
