# Block 2: 내 Skill 만들기


## EXPLAIN

## [2] 내 Skill 만들기
═══════════════════════════════

이제 본인의 반복 업무를 Skill로 만들어봅시다.

### SKILL.md 작성법

```yaml
---
name: weekly-report
description: 주간보고서 자동 생성. `/weekly-report` 명령으로 시작.
---
```

💡 이름은 영어로 짓는 것을 권장합니다. 터미널에서 입력이 편합니다.

그 아래에 Claude에게 할 지시를 적습니다:
- 뭘 해야 하는지
- 어떤 순서로
- 어떤 형식으로 결과를 내는지


### references/ 파일 작성법

구체적인 지시, 예시, 템플릿을 넣습니다.
SKILL.md에서 "references/파일명.md를 읽어라"고 지시합니다.


### EXEM 업무 Skill 예시

  📋 /weekly-report (주간보고)
  ─────────────────────────────
  이번 주 작업 파일들을 읽어서
  주간보고서 양식으로 정리

  📝 /meeting-notes (회의록정리)
  ─────────────────────────────
  회의 내용을 구조화하여
  액션 아이템 + 결정 사항 + 다음 단계로 정리

  🔍 /competitor-check (경쟁사체크)
  ─────────────────────────────
  경쟁사 키워드로 관련 정보를 수집하고
  요약 보고서 작성


### 내 Skill 아이디어 잡기

내가 매주/매일 반복하는 업무를 생각해보세요:

  ✅ 매주 쓰는 보고서가 있다
  ✅ 같은 형식으로 정리하는 문서가 있다
  ✅ 매번 비슷한 요청을 Claude에게 한다
  ✅ 팀원들에게도 공유하고 싶은 워크플로우가 있다

하나라도 해당되면 Skill로 만들 수 있습니다!

## EXECUTE

### 내 Skill 직접 만들기

**1단계: 주제 선택**
어떤 반복 업무를 Skill로 만들지 정합니다.

💡 Claude에게 도움을 요청해도 됩니다:
```
나는 EXEM에서 [업무]를 하고 있어.
매주 반복하는 업무 중에 Skill로 만들면 좋을 것을 추천해줘.
```

**2단계: SKILL.md 작성**
Claude에게 이렇게 요청합니다:
```
.claude/skills/[내스킬이름]/SKILL.md를 만들어줘.

이 Skill은 [업무 설명]을 자동화하는 거야.
/[내스킬이름]을 입력하면:
1. [첫 번째 단계]
2. [두 번째 단계]
3. [결과 형식]
```

**3단계: references/ 작성**
필요하면 상세 지시를 references/ 에 추가합니다:
```
.claude/skills/[내스킬이름]/references/template.md를 만들어줘.
[상세 템플릿이나 예시]를 넣어줘.
```

**4단계: 테스트**
만든 Skill을 실행해봅니다:
```
/[내스킬이름]
```

💡 처음부터 완벽할 필요 없습니다.
   실행해보고 수정하는 과정이 자연스럽습니다.
