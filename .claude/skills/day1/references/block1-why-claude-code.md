## [1] 왜 Claude Code인가
═══════════════════════════════

"ChatGPT 웹이나 Claude 웹도 잘 되는데, 왜 굳이 터미널에서?"
좋은 질문입니다. 솔직하게 이야기합시다.

### 먼저, 인정할 것

2026년 기준, 웹 AI도 엄청나게 발전했습니다:
- ChatGPT, Claude 웹 모두 MCP 지원 (외부 도구 연결)
- 파일 업로드, 코드 실행, 이미지 분석 다 됨
- Manus 같은 범용 에이전트는 웹 앱까지 만들어줌

그럼에도 Claude Code를 쓰는 이유는 "더 똑똑해서"가 아닙니다.

### 진짜 차이: 내 맥락이 쌓이는 AI

  웹 AI                           Claude Code
  ─────────────                  ─────────────
  대화할 때만 똑똑함              세팅이 쌓일수록 똑똑해짐
  내 맥락을 매번 설명              CLAUDE.md에 저장해두면 기억
  매번 비슷한 요청 반복           /명령어 한마디로 끝
  다른 사람의 노하우 못 씀        플러그인으로 가져다 쓸 수 있음

비유하면 이렇습니다:

  🌐 웹 AI = 똑똑한 프리랜서
     실력은 좋지만, 매번 새로 브리핑해야 함

  💻 Claude Code = 내 세팅이 된 AI
     CLAUDE.md에 내 규칙이 있고, Skill로 반복 업무가 자동화되어 있음

### 핵심 3가지

  1. CLAUDE.md — AI의 업무 매뉴얼. 내 규칙과 선호를 저장
  2. Skill (/명령어) — 반복 업무를 한마디로 자동화
  3. 플러그인 생태계 — 다른 사람이 만든 워크플로우를 설치해서 쓸 수 있음

이 세 가지는 웹 AI에서는 아직 없거나 미성숙한 영역입니다.
이번 교육에서 이 세 가지를 직접 만들어봅니다.
