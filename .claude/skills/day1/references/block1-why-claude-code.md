# Block 1: 왜 Claude Code인가


## EXPLAIN

## [1] 왜 Claude Code인가
═══════════════════════════════

"ChatGPT 웹이나 Claude 웹도 잘 되는데, 왜 굳이 터미널에서?"
좋은 질문입니다. 직접 비교해보면 답이 나옵니다.

  🌐 웹 Chat                     💻 Claude Code
  ─────────────                  ─────────────
  파일을 복사해서 붙여넣기        내 파일을 직접 읽고 수정
  AI가 알아서 기억 (통제 어려움)  CLAUDE.md로 내가 직접 관리 + 팀 공유
  매번 비슷한 요청 반복           /명령어(Skill)로 자동화
  내 프로젝트 맥락 모름           프로젝트 전체를 이해

비유하면 이렇습니다:

  📞 웹 Chat = 전화 상담
     내가 상황을 매번 설명해야 하고, 끊으면 잊어버림

  🏢 Claude Code = 회사에 들어와서 같이 일하는 동료
     내 파일을 보고, 내 규칙을 기억하고, 도구도 직접 씀

Lenny Rachitsky(실리콘밸리 뉴스레터 저자)는 이렇게 표현했습니다:
"Claude Code를 'Claude Local' 또는 'Claude Agent'로 생각하라.
브라우저에서 벗어나 내 환경에서 직접 일하는 AI다."

핵심: Claude Code는 더 똑똑한 AI가 아니라, **더 많은 맥락을 가진 AI**입니다.

## EXECUTE

### 직접 비교 체험

같은 요청을 웹과 Claude Code에서 해보면 차이를 체감할 수 있습니다.

**체험 1:** 아래를 그대로 입력해보세요:
```
이 프로젝트에 어떤 파일이 있는지 알려줘
```
→ Claude Code는 직접 파일 시스템을 읽어서 답합니다.
→ 웹 Chat이었다면? "파일 목록을 붙여넣어 주세요"라고 했을 겁니다.

**체험 2:** 이것도 해보세요:
```
README.md를 읽고 3줄로 요약해줘
```
→ Claude Code는 README.md를 직접 열어서 읽고 요약합니다.
→ 웹 Chat이었다면? 파일 내용을 복사해서 붙여넣어야 합니다.

💡 이 차이가 Day 2~5에서 점점 더 커집니다.
   CLAUDE.md로 맥락을 쌓고, MCP로 도구를 연결하고,
   Skill로 반복 작업을 자동화하면
   웹 Chat으로는 어려운 일들을 하게 됩니다.
