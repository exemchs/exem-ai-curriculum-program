## [4] CLAUDE.md — 내 비서 세팅
═══════════════════════════════

세 번째 레벨입니다.
"매번 프롬프트를 잘 쓰는 것보다, 시스템을 설계하는 게 더 중요하다."

CO-STAR(Lv.1)과 CRIT(Lv.2)는 매번 잘 써야 합니다.
CLAUDE.md는 한 번 세팅하면 매번 자동으로 적용됩니다.

  CO-STAR   매번 구조를 일일이 작성
  CRIT      매번 "질문 먼저 해줘"를 타이핑
  CLAUDE.md 한 번 적으면 자동으로 적용 ✨

Phil Schmid(Hugging Face 엔지니어)의 말:
"대부분의 에이전트 실패는 모델 실패가 아니라 컨텍스트 실패다."

CLAUDE.md는 Claude Code의 **영구 기억**입니다.
한 번 적으면 매번 자동으로 읽습니다.

비유: 신입 비서에게 첫 날 주는 업무 매뉴얼

  📋 매뉴얼 없는 비서            📖 매뉴얼 있는 비서
  ─────────────                 ─────────────
  매번 "저는 김OO이고..."       이미 알고 있음
  내 스타일을 모름               선호 스타일대로 작성
  같은 실수 반복                 실수 방지 규칙 적용

Boris Cherny(Claude Code 창시자)의 조언:
"실수를 발견하면 CLAUDE.md에 추가하세요. 같은 실수를 반복하지 않게."


### CLAUDE.md에 넣으면 좋은 것

  ✅ 이름 + 역할       "나는 [이름], EXEM [팀명]에서 [업무]"
  ✅ 응답 규칙         "한국어로 답해", "표를 활용해", "간결하게"
  ✅ EXEM 맥락         "EXEM은 IT 모니터링 기업, 주력 제품은..."
  ✅ 선호 규칙         "하지 말 것: 영어로 답하지 마"
  ✅ 실수 방지 규칙    "항상 출처를 밝혀"


### CLAUDE.md에 넣으면 안 되는 것

  ❌ 비밀번호, API 키 등 민감 정보
  ❌ 너무 긴 문서 (200줄 이하 권장)
  ❌ 모호한 지시 ("잘 해줘" 같은)


### 좋은 CLAUDE.md의 원칙

  1. 200줄 이하로 유지
  2. 검증 가능한 구체적 지시 ("간결하게" ✅ / "잘" ❌)
  3. 정기적으로 정리 (안 쓰는 규칙 삭제)

## EXECUTE

### 내 CLAUDE.md 작성하기

지금 직접 CLAUDE.md를 만들어봅시다!

Claude에게 이렇게 말해보세요:

```
내 CLAUDE.md를 만들어줘. 아래 내용을 포함해서:

- 이름: [내 이름]
- 역할: EXEM [팀명]에서 [주요 업무]를 담당
- 응답 규칙:
  - 한국어로 답할 것
  - 간결하게, 표를 활용할 것
  - [내가 원하는 규칙 1가지 추가]
- 하지 말 것:
  - [내가 싫어하는 것 1가지]
```

작성이 끝나면 확인해봅시다:

```
내가 누군지 알아?
```

→ Claude가 CLAUDE.md에 적은 내 정보를 기억하고 있는지 확인!
