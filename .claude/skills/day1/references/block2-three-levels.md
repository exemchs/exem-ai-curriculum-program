## [2] AI에게 잘 시키는 3가지 레벨
═══════════════════════════════

AI에게 일을 시키는 방법에는 수준이 있습니다.
레벨이 올라갈수록 결과의 품질이 달라집니다.


### 🔹 Lv.1  구조화된 요청 (CO-STAR)

"대충 말하지 말고, 구조를 갖춰서 말하라."

CO-STAR 프레임워크는 요청을 6가지로 나눕니다:

  C  Context    맥락       "EXEM은 IT 모니터링 기업이고..."
  O  Objective  목표       "경쟁사 분석 보고서를 만들어줘"
  S  Style      스타일     "표 형식으로, 간결하게"
  T  Tone       톤        "비즈니스 보고서 톤"
  A  Audience   대상       "CX그룹 팀장이 읽을 예정"
  R  Response   응답형식   "마크다운 표로"

💡 예시: EXEM 경쟁사 분석 요청을 CO-STAR로 구조화하면:
   "EXEM은 IT 모니터링 기업이야(C). 경쟁사 Whatap, Datadog과 비교 분석해줘(O).
    표 형식으로 간결하게(S), 비즈니스 톤으로(T), 경영진 보고용(A),
    마크다운 표로 출력해줘(R)."

### 🔷 Lv.2  AI가 먼저 질문하게 (CRIT Interview)

"내가 다 지시하는 게 아니라, AI에게 역질문을 시켜라."

이런 식으로 시작합니다:
```
나는 [업무 주제]에 대한 보고서를 쓰려 해.
바로 쓰지 말고, 시작하기 전에 핵심 질문 5개를 먼저 던져줘.
```

왜 이게 더 좋을까요?

  Lv.1  내가 모든 걸 미리 생각해서 지시
        → 내가 빠뜨린 건 AI도 빠뜨림

  Lv.2  AI가 먼저 빠진 부분을 질문
        → 내가 생각 못한 것까지 커버

비유: 컨설턴트를 고용했을 때
- 나쁜 컨설턴트: 바로 보고서를 씀 → 방향이 어긋남
- 좋은 컨설턴트: 첫 미팅에서 질문부터 → 정확한 결과

───────────────────────────────


### 🔶 Lv.3  시스템에 새기기 (CLAUDE.md)

"매번 프롬프트를 잘 쓰는 것보다, 시스템을 설계하는 게 더 중요하다."

Lv.1과 Lv.2는 매번 잘 써야 합니다.
Lv.3은 한 번 세팅하면 매번 적용됩니다.

  Lv.1  매번 CO-STAR를 일일이 작성
  Lv.2  매번 "질문 먼저 해줘"를 타이핑
  Lv.3  CLAUDE.md에 적어두면 자동으로 적용 ✨

Phil Schmid(Hugging Face 엔지니어)의 말:
"대부분의 에이전트 실패는 모델 실패가 아니라 컨텍스트 실패다."

→ 다음 Block에서 직접 CLAUDE.md를 만들어봅니다.

## EXECUTE

### Lv.1 실습: CO-STAR로 요청해보기

아래를 참고해서 본인의 업무에 맞는 CO-STAR 요청을 만들어보세요:

```
[C] 나는 EXEM에서 [업무]를 하고 있어.
[O] [원하는 결과]를 만들어줘.
[S] [형식: 표/리스트/문서].
[T] [톤: 비즈니스/캐주얼/설명적].
[A] [누가 읽을지].
[R] [출력 형식].
```

직접 타이핑해서 Claude에게 보내보세요!

### Lv.2 실습: 역질문 패턴 체험

이번엔 같은 주제로 이렇게 요청해보세요:

```
나는 [위에서 작성한 주제]에 대해 작업하려 해.
바로 시작하지 말고, 더 좋은 결과를 위해
핵심 질문 5개를 먼저 던져줘.
```

💡 Lv.1과 Lv.2의 결과를 비교해보세요.
   AI가 질문한 후 작업한 결과가 어떻게 다른지 체감할 수 있습니다.
