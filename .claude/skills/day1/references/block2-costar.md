## EXPLAIN

## [2] CO-STAR — 구조화된 요청
═══════════════════════════════

AI에게 일을 시키는 방법에는 수준이 있습니다.
오늘 3가지를 하나씩 배웁니다. 먼저 첫 번째.

"대충 말하지 말고, 구조를 갖춰서 말하라."

CO-STAR 프레임워크는 요청을 6가지로 나눕니다:

  C  Context    맥락       "EXEM은 IT 모니터링 기업이고..."
  O  Objective  목표       "경쟁사 분석 보고서를 만들어줘"
  S  Style      스타일     "표 형식으로, 간결하게"
  T  Tone       톤        "비즈니스 보고서 톤"
  A  Audience   대상       "CX그룹 팀장이 읽을 예정"
  R  Response   응답형식   "마크다운 표로"

### Before / After

  😕 Before: "경쟁사 분석해줘"
  → AI가 뭘 비교할지, 누가 읽을지, 어떤 형식인지 모름
  → 결과가 내가 원하는 것과 다를 확률 높음

  🎯 After (CO-STAR):
  "EXEM은 IT 모니터링 기업이야(C).
   경쟁사 Whatap, Datadog과 비교 분석해줘(O).
   표 형식으로 간결하게(S), 비즈니스 톤으로(T),
   경영진 보고용(A), 마크다운 표로 출력해줘(R)."
  → AI가 정확히 뭘 해야 하는지 알고, 첫 결과부터 쓸만함

💡 CO-STAR 6가지를 다 채울 필요는 없습니다.
   중요한 건 "대충 말하기"에서 "구조를 갖춰 말하기"로 바꾸는 것입니다.

## EXECUTE

### CO-STAR로 요청해보기

아래를 참고해서 본인의 업무에 맞는 CO-STAR 요청을 만들어보세요:

```
[C] 나는 EXEM에서 [업무]를 하고 있어.
[O] [원하는 결과]를 만들어줘.
[S] [형식: 표/리스트/문서].
[T] [톤: 비즈니스/캐주얼/설명적].
[A] [누가 읽을지].
[R] [출력 형식].
```

직접 타이핑해서 Claude에게 보내보세요!

💡 CO-STAR 6가지를 다 채울 필요는 없습니다.
   중요한 건 "대충 말하기"에서 "구조를 갖춰 말하기"로 바꾸는 것입니다.
