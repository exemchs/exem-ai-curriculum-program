# Block 2: 최종 실행 + 검증

## EXPLAIN

### 만든 스킬 실행하기

스킬은 `/스킬이름`으로 실행합니다.

```
/my-sync
```

실행하면 Claude가 SKILL.md를 읽고, 거기에 적힌 대로 동작합니다.

### 잘 동작하는지 확인하기

| 확인 항목 | 방법 |
|----------|------|
| 스킬이 인식되는가? | `/` 입력 후 자동완성 목록에 나오는지 확인 |
| SKILL.md를 읽는가? | 실행 시 기대한 흐름대로 진행되는지 확인 |
| 출력이 올바른가? | 결과 문서가 템플릿대로 생성되었는지 확인 |
| 에러가 있는가? | 에러 메시지가 나오면 Claude에게 수정 요청 |

### 안 되면?

Claude에게 바로 물어보세요:

```
방금 /my-sync 실행했는데 [문제 상황]. 원인이 뭘까?
```

Claude가 SKILL.md를 읽고 문제를 진단해줍니다.

## EXECUTE

1. **스킬 실행**: 만든 스킬을 `/스킬이름`으로 실행하세요
2. **결과 확인**: 출력이 기대한 대로 나왔는지 확인하세요
3. **문제 수정**: 문제가 있으면 Claude에게 수정 요청하세요
4. **재실행**: 수정 후 다시 실행해서 확인하세요

> 한 번에 완벽할 필요 없습니다. 실행 → 확인 → 수정 → 재실행을 반복하는 것이 스킬 개발의 정상적인 과정입니다.
