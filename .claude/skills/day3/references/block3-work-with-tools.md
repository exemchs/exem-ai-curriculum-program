## [3] Skill & 플러그인 — 다른 사람의 노하우를 가져다 쓰기
═══════════════════════════════

MCP로 도구를 연결하는 것도 좋지만,
Claude Code의 진짜 강점은 **Skill과 플러그인 생태계**입니다.

### Skill이란?

반복 업무를 `/명령어`로 저장하는 것입니다.

  Skill 없을 때                   Skill 있을 때
  ─────────────                  ─────────────
  매번 긴 요청을 타이핑           `/weekly-report` 한마디
  매번 형식을 지정                 저장된 형식대로 자동
  사람마다 결과가 다름             누가 해도 같은 품질

사실 여러분은 이미 Skill을 체험하고 있습니다!
`/day3`를 입력한 것 자체가 Skill 호출입니다.

### 플러그인 = 다른 사람이 만든 Skill을 가져다 쓰는 것

앱스토어에서 앱을 설치하듯,
다른 사람이 만든 워크플로우를 내 Claude Code에 설치할 수 있습니다.

  📱 앱스토어에서 앱 설치         🔌 Claude Code에 플러그인 설치
  ─────────────                  ─────────────
  다른 사람이 만든 앱             다른 사람이 만든 워크플로우
  설치하면 바로 사용              설치하면 /명령어로 바로 사용
  내가 만들 필요 없음             내가 만들 필요 없음

### 실제 사례: GSD (Get Shit Done)

GitHub에서 인기 있는 Claude Code 플러그인 중 하나입니다.

  GSD가 하는 일:
  ─────────────────────────────
  프로젝트를 분석해서 로드맵을 자동 생성
  각 단계를 계획 → 실행 → 검증까지 관리
  진행 상황을 추적하고 다음 할 일을 안내

  사용법: /gsd:new-project, /gsd:progress 같은 명령어

이런 플러그인이 수백 개 있습니다.
코딩, 문서 작성, 프로젝트 관리, 디버깅, 코드 리뷰...
누군가가 이미 만들어놓은 워크플로우를 가져다 쓸 수 있습니다.

### Day 1에서 배운 "세팅이 쌓이는 AI"의 완성

  Day 1: CLAUDE.md — AI의 업무 매뉴얼 (기억)
  Day 3: MCP — AI에게 도구 연결 (눈과 손)
  Day 3: Skill/플러그인 — 반복 업무 자동화 (습관)

  이 세 가지가 합쳐지면
  웹 AI로는 불가능한, 나만의 AI 시스템이 됩니다.
